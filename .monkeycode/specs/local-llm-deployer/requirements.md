# 需求文档

## 引言

本工具是一款本地一键部署开源大模型的桌面应用程序，帮助用户快速部署、管理和切换本地大语言模型。系统通过智能检测本地硬件配置，自动推荐最适合的模型，提供可视化的下载安装体验，并生成可用的 API 地址、密钥、模型 ID 和提供商信息。

## 词汇表

- **Ollama**: 本地大模型推理引擎，提供 REST API 接口
- **API Provider**: 提供商标识，用于区分不同模型服务来源
- **Model ID**: 模型唯一标识符，格式为 `model:tag`
- **系统配置**: 本地硬件资源，包括 GPU/内存/磁盘/CPU
- **部署状态**: 模型当前运行状态，包含已停止/启动中/运行中/错误
- **下载任务**: 模型文件下载任务，包含进度、速度、状态信息

## 需求

### 需求 1：系统配置检测

**用户故事：** AS 开发者，我想要系统自动检测本地硬件配置，以便智能推荐适合的模型。

#### Acceptance Criteria

1. WHEN 用户启动应用程序，系统 SHALL 自动检测并显示 CPU 型号、核心数、物理内存总量
2. WHEN 用户启动应用程序，系统 SHALL 自动检测 GPU 信息，包括 GPU 型号、显存大小、驱动版本
3. WHEN 用户启动应用程序，系统 SHALL 自动检测磁盘空间，包括总容量和可用空间
4. WHEN 系统配置发生变更（外接显示器/扩展坞），系统 SHALL 自动重新检测并更新显示
5. IF 检测失败，系统 SHALL 显示具体错误信息并提供手动输入配置选项

### 需求 2：智能模型推荐

**用户故事：** AS 普通用户，我想要系统根据我的硬件配置推荐可运行的模型，省去手动选择的麻烦。

#### Acceptance Criteria

1. WHEN 系统完成配置检测，系统 SHALL 根据硬件资源（GPU 显存/物理内存）生成可运行模型列表
2. WHEN 系统生成推荐列表，系统 SHALL 按模型参数量从大到小排序，明确标注每个模型的最低配置要求
3. WHEN 系统生成推荐列表，系统 SHALL 显示每个模型的预估内存占用和推理速度评级
4. IF 用户硬件配置过低无法运行任何模型，系统 SHALL 提供详细说明并建议硬件升级路径
5. IF 用户手动搜索模型，系统 SHALL 实时验证模型与当前配置兼容性并显示警告

### 需求 3：开源模型库

**用户故事：** AS 开发者，我想要查看所有支持的免费开源模型列表及其详细信息。

#### Acceptance Criteria

1. WHEN 用户打开模型库页面，系统 SHALL 展示支持的模型列表，包括模型名称、参数量、量化选项、下载地址
2. WHEN 用户选择模型，系统 SHALL 显示模型的详细描述、版本标签、文件大小、依赖要求
3. WHEN 用户搜索模型，系统 SHALL 支持按名称/参数量/提供商/标签筛选模型
4. IF 模型因版权或法律原因不可用，系统 SHALL 明确标注并说明原因
5. IF 模型文件存储位置变更，系统 SHALL 保持模型元数据引用正确性

### 需求 4：模型下载管理

**用户故事：** AS 用户，我想要可视化下载模型并显示实时进度，支持自定义存储位置。

#### Acceptance Criteria

1. WHEN 用户发起模型下载，系统 SHALL 显示下载速度、已下载大小、总大小、预估剩余时间
2. WHEN 用户下载模型，系统 SHALL 在界面上显示进度条，支持暂停、恢复、取消操作
3. WHEN 用户点击暂停，系统 SHALL 保存下载进度并支持后续恢复
4. IF 下载中断（网络故障/磁盘满），系统 SHALL 自动保存进度并在网络恢复后继续下载
5. IF 下载完成，系统 SHALL 验证文件完整性并通知用户
6. IF 用户未设置存储位置，系统 SHALL 使用默认路径 `~/.ollama/models`

### 需求 5：自定义存储位置

**用户故事：** AS 高级用户，我想要自定义模型的下载和存储位置，以便管理磁盘空间。

#### Acceptance Criteria

1. WHEN 用户首次使用，系统 SHALL 引导用户设置默认模型存储路径
2. WHEN 用户进入设置页面，系统 SHALL 允许修改模型存储路径并实时显示已用空间
3. WHEN 用户修改存储路径，系统 SHALL 验证路径有效性并自动迁移已下载模型文件
4. IF 目标路径磁盘空间不足，系统 SHALL 警告用户并阻止移动操作
5. IF 存储路径包含特殊字符或权限不足，系统 SHALL 显示错误提示并要求重新选择

### 需求 6：Ollama 引擎管理

**用户故事：** AS 用户，我想要一键安装和配置 Ollama 推理引擎，系统自动处理依赖。

#### Acceptance Criteria

1. WHEN 用户首次启动且未安装 Ollama，系统 SHALL 自动检测并提示安装
2. WHEN 用户确认安装 Ollama，系统 SHALL 下载并安装对应平台版本，安装完成后显示版本信息
3. WHEN Ollama 已安装但版本过旧，系统 SHALL 提示用户升级并支持一键升级
4. IF Ollama 安装失败，系统 SHALL 检测问题原因（权限/磁盘/网络）并提供具体解决方案
5. IF Ollama 服务异常退出，系统 SHALL 自动重启服务并通知用户

### 需求 7：模型部署启动

**用户故事：** AS 用户，我想要一键启动已下载的模型，无需手动执行命令行。

#### Acceptance Criteria

1. WHEN 用户点击启动按钮且模型已下载完成，系统 SHALL 执行 `ollama run` 命令启动模型
2. WHEN 模型启动成功，系统 SHALL 实时显示模型加载状态和显存占用
3. WHEN 模型正在运行时，系统 SHALL 在界面上显示运行指示器和基本信息
4. IF 模型启动失败（显存不足/依赖缺失），系统 SHALL 分析失败原因并提供具体错误信息和解决建议
5. IF 同一模型已在运行，系统 SHALL 提示用户而非重复启动

### 需求 8：多模型管理

**用户故事：** AS 开发者，我想要同时管理多个模型，支持快速切换不同模型。

#### Acceptance Criteria

1. WHEN 用户下载多个模型，系统 SHALL 在模型列表中显示所有已下载模型及其状态
2. WHEN 用户需要切换模型，系统 SHALL 支持热切换，无需重启 Ollama 服务
3. WHEN 用户切换模型，系统 SHALL 记录切换历史并支持快速回退到上一个模型
4. IF 用户尝试运行与当前模型不兼容的新请求，系统 SHALL 提示用户先切换模型
5. IF 系统资源不足以支持当前模型，系统 SHALL 建议用户停止当前模型后启动新模型

### 需求 9：模型停止管理

**用户故事：** AS 用户，我想要停止正在运行的模型以释放系统资源。

#### Acceptance Criteria

1. WHEN 用户点击停止按钮，系统 SHALL 向 Ollama 发送停止信号并等待模型优雅退出
2. WHEN 模型超过 30 秒未响应停止信号，系统 SHALL 强制终止进程
3. WHEN 模型成功停止，系统 SHALL 更新界面状态并显示释放的资源量
4. IF 停止操作失败，系统 SHALL 记录错误日志并提示用户手动处理

### 需求 10：运行状态监控

**用户故事：** AS 用户，我想要实时监控当前模型的运行状态和资源使用情况。

#### Acceptance Criteria

1. WHEN 模型运行中，系统 SHALL 实时显示 CPU 使用率、内存占用、GPU 使用率、显存占用
2. WHEN 模型处理请求时，系统 SHALL 显示当前请求的处理进度和预估响应时间
3. WHEN 系统资源紧张（显存 > 90%），系统 SHALL 显示警告信息
4. IF 模型响应时间异常延长（> 60 秒），系统 SHALL 提示用户可能存在性能问题
5. IF Ollama 服务意外终止，系统 SHALL 立即更新状态并尝试自动重启

### 需求 11：API 信息生成

**用户故事：** AS 开发者，我想要获取模型的 API 地址、密钥、模型 ID 和提供商信息。

#### Acceptance Criteria

1. WHEN 模型成功启动，系统 SHALL 自动生成 API 地址，格式为 `http://localhost:11434`
2. WHEN 用户请求 API 信息，系统 SHALL 生成完整的 API 端点，包括 `/api/generate`、`/api/chat`、`/api/tags`
3. WHEN 用户请求 API 密钥，系统 SHALL 生成符合 Ollama 规范的本地 API 密钥（基于本地认证）
4. WHEN 用户请求模型 ID，系统 SHALL 显示当前运行模型的完整标识符（格式：`model:tag`）
5. WHEN 用户请求提供商信息，系统 SHALL 显示 `provider: "ollama"` 和版本信息
6. IF Ollama 未运行或模型未启动，系统 SHALL 返回空值并提示用户启动模型

### 需求 12：API 可用性验证

**用户故事：** AS 开发者，我想要验证生成的 API 信息是否真实可用，确保可以正常调用。

#### Acceptance Criteria

1. WHEN 用户点击测试按钮，系统 SHALL 向 `http://localhost:11434/api/tags` 发送验证请求
2. WHEN API 响应成功（HTTP 200），系统 SHALL 显示绿色标识表示 API 可用
3. WHEN API 响应失败（超时/连接拒绝），系统 SHALL 显示红色标识并列出具体错误原因
4. WHEN 验证通过后，系统 SHALL 执行一次完整的生成请求测试，验证模型实际推理能力
5. IF 测试请求失败（CUDA 错误/内存不足），系统 SHALL 详细记录错误信息并提供解决建议
6. IF API 密钥验证失败，系统 SHALL 提示用户检查 Ollama 认证配置

### 需求 13：配置导出功能

**用户故事：** AS 开发者，我想要导出当前模型的完整配置信息，方便在其他环境复现。

#### Acceptance Criteria

1. WHEN 用户点击导出配置，系统 SHALL 生成包含 API 地址、模型 ID、提供商信息的 JSON 文件
2. WHEN 用户导出配置，系统 SHALL 支持复制到剪贴板或保存到本地文件
3. IF 用户需要共享配置，系统 SHALL 生成不含敏感信息的公开配置版本

### 需求 14：配置导入功能

**用户故事：** AS 开发者，我想要导入已有的模型配置，快速恢复工作环境。

#### Acceptance Criteria

1. WHEN 用户点击导入配置，系统 SHALL 解析用户提供的 JSON 配置文件
2. WHEN 配置文件格式正确，系统 SHALL 验证模型 ID 是否已下载，未下载时提示用户先下载模型
3. IF 配置文件格式错误或缺少必要字段，系统 SHALL 显示具体错误信息并拒绝导入
4. IF 导入成功，系统 SHALL 自动填充 API 信息并验证可用性

### 需求 16：本地 API 认证

**用户故事：** AS 用户，我想要在不增加复杂度的情况下使用 Ollama 原生 API。

#### Acceptance Criteria

1. WHEN 用户使用默认配置，系统 SHALL 使用 Ollama 原生 API，无需额外认证
2. WHEN Ollama 升级后，系统 SHALL 保持 API 兼容性，无需重新配置
3. IF 用户需要增强安全性，系统 SHALL 提供通过环境变量配置 API 密钥的选项

### 需求 17：完全离线支持

**用户故事：** AS 用户，我想要在完全没有网络的环境中使用已下载的模型。

#### Acceptance Criteria

1. WHEN 模型已下载完成且 Ollama 已安装，系统 SHALL 支持完整的本地模型启动和推理
2. WHEN 系统启动时检测不到网络，系统 SHALL 自动切换到离线模式并加载已下载模型列表
3. WHEN 用户在离线模式下，系统 SHALL 显示网络状态图标并禁止下载新模型
4. IF 用户尝试下载模型且网络不可用，系统 SHALL 提示用户检查网络连接并显示已下载模型列表
5. IF 用户导入已下载的模型文件（手动复制），系统 SHALL 自动识别并添加到模型列表

### 需求 18：自定义模型源

**用户故事：** AS 高级用户，我想要添加第三方或私有模型源以获取更多模型选择。

#### Acceptance Criteria

1. WHEN 用户进入设置页面，系统 SHALL 提供添加自定义模型源的选项
2. WHEN 用户添加自定义源，系统 SHALL 验证源地址可用性并获取模型列表
3. WHEN 用户添加自定义源，系统 SHALL 支持 HuggingFace、ModelScope 等常见模型平台格式
4. IF 自定义源 URL 格式错误或不可访问，系统 SHALL 显示错误信息并允许用户重试或移除
5. IF 自定义源包含与官方源相同的模型，系统 SHALL 允许用户选择使用哪个源

### 需求 19：界面国际化

**用户故事：** AS 普通用户，我想要使用母语界面，降低使用门槛。

#### Acceptance Criteria

1. WHEN 用户首次启动，系统 SHALL 自动检测系统语言并显示对应界面
2. WHEN 用户进入设置页面，系统 SHALL 允许手动切换界面语言
3. WHEN 系统支持的语言包括：简体中文、English
4. IF 用户选择的语言缺少翻译，系统 SHALL 回退到 English 显示
